` Board of Issues API

📋 Board of Issues API

Добро пожаловать в API сервиса управления задачами и досками!

🔐 Аутентификация: Большинство эндпоинтов требуют Bearer токен в заголовке Authorization: Bearer <ваш_токен>

📌 Формат ответов: Все ответы приходят в формате JSON. В случае ошибки будет поле {"error": "описание"}.

👤 Пользователи

POST /register
Регистрация нового пользователя
Тело запроса: { "name": "Имя", "email": "user@mail.com", "login": "login123", "password": "1234567" }
POST /login
Вход в систему. Можно использовать email или логин
Вариант 1 (по email): { "email": "user@mail.com", "password": "1234567" }
Вариант 2 (по логину): { "login": "login123", "password": "1234567" }
PATCH /api/users/name
Изменение имени пользователя (требуется токен)
Тело запроса: { "name": "Новое имя" }
PATCH /api/users/password
Изменение пароля (требуется токен)
Тело запроса: { "password": "новый_пароль" }
PATCH /api/users/email
Изменение email (требуется токен)
Тело запроса: { "email": "new@mail.com" }
POST /api/users
Подключение пользователя к доске (требуется токен)
Тело запроса: { "desk_id": 123, "password": "пароль_доски" }
⚠️ Важно: Пароль доски обязателен для подключения!

📊 Доски

POST /api/desks
Создание новой доски (требуется токен)
Тело запроса: { "name": "Моя доска", "password": "1234567" }
GET /api/desks
Получение списка всех досок пользователя (требуется токен)
Заголовки: Authorization: Bearer <токен>
PATCH /api/desks/{id}/name
Изменение названия доски (только владелец, требуется токен)
Параметры пути: id - идентификатор доски
Тело запроса: { "name": "Новое название" }
PATCH /api/desks/{id}/password
Изменение пароля доски (только владелец, требуется токен)
Параметры пути: id - идентификатор доски
Тело запроса: { "password": "новый_пароль" }
PATCH /api/desks/{id}/owner
Смена владельца доски (только текущий владелец, требуется токен)
Параметры пути: id - идентификатор доски
Тело запроса: { "new_owner_id": 456 }
DELETE /api/desks/{id}
Удаление доски (только владелец, требуется токен)
Параметры пути: id - идентификатор доски

✅ Задачи

POST /api/tasks
Создание задачи (требуется токен)
Тело запроса: { "title": "Задача", "description": "Описание", "desk_id": 123 }
GET /api/tasks/{deskId}
Получение всех задач доски (требуется токен)
Параметры пути: deskId - идентификатор доски
GET /api/tasks?done=true&desk_id=123
Получение задач с фильтрацией (требуется токен)
Параметры запроса:
done (опционально): true/false - фильтр по статусу выполнения
desk_id (обязательно): идентификатор доски
PATCH /api/tasks/{id}/complete
Отметить задачу как выполненную (требуется токен)
Параметры пути: id - идентификатор задачи
PATCH /api/tasks/{id}/time
Добавить время к задаче (требуется токен)
Параметры пути: id - идентификатор задачи
Тело запроса: { "hours": 2 }
PATCH /api/tasks/{id}/description
Изменить описание задачи (требуется токен)
Параметры пути: id - идентификатор задачи
Тело запроса: { "description": "Новое описание" }
DELETE /api/tasks/{id}
Удаление задачи (требуется токен)
Параметры пути: id - идентификатор задачи

📌 Примеры использования с cURL

# 1. Регистрация
curl -X POST http://localhost:8080/register \
-H "Content-Type: application/json" \
-d '{"name":"Иван","email":"ivan@mail.com","login":"ivan123","password":"123456"}'

# 2. Вход и сохранение токена
curl -X POST http://localhost:8080/login \
-H "Content-Type: application/json" \
-d '{"email":"ivan@mail.com","password":"123456"}'

# 3. Подключение к доске (с паролем!)
curl -X POST http://localhost:8080/api/users \
-H "Authorization: Bearer ВАШ_ТОКЕН" \
-H "Content-Type: application/json" \
-d '{"desk_id": 1, "password": "пароль_доски"}'
