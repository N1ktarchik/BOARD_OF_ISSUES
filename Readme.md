# 📋 Board of Issues 2.0

Добро пожаловать во вторую версию **Board of Issues**. Эта версия представляет собой полный архитектурный рефакторинг: переход от монолитной структуры к модульной «чистой архитектуре» (Clean Architecture), удовлетворяющей принципам SOLID.

**Цель рефакторинга:** Масштабируемость, типизация и документация API уровня production.

## 🛠 Технологический стек

* **Язык:** Go (Golang) 1.25.1
* **База данных:** PostgreSQL + `pgx` 
* **Роутер:** `gorilla/mux`
* **Логирование:** `slog` 
* **Документация:** Swagger (OpenAPI 2.0) через `swag`
* **Миграции:** `golang-migrate/migrate` (управление схемой БД)
* **Аутентификация:** JWT (JSON Web Tokens)
* **Качество кода:** `golangci-lint` 
* **Тестирование:** `testing` + `mockgen` (Unit-тесты с автоматической генерацией моков)
* **CI/CD:** GitHub Actions (автоматизация тестов, линтинга и сборки образов)
* **Контейнеризация:** Docker & Docker Compose

## 🏗 Что изменилось в архитектуре (v2 vs v1)

В отличие от первой версии, в **v2** внедрено строгое разделение ответственности:
* **Domain Layer:** Чистая бизнес-логика и сущности. Никакой зависимости от внешних библиотек.
* **Transport Layer (HTTP):** Изолированная обработка HTTP-запросов и ответов.
* **DTO (Data Transfer Objects):** Входящие данные проходят строгую валидацию и типизацию перед тем, как попасть в сервис.
* **Repository Pattern:** Работа с БД скрыта за интерфейсами. Используется пул соединений `pgx` для эффективности.
* **Централизованная обработка ошибок:** Все ответы API имеют единый формат, что упрощает жизнь фронтенд-разработчикам.

**Также добавлены и улучшены:**
* **Миграции:** Удобное управление версиями схемы БД.
* **Документация:** Интерактивный Swagger доступен "из коробки".
* **Логирование:** Структурированный вывод для упрощения отладки.
* **Makefile:** Автоматизация рутинных задач одной командой.
* **Линтинг:** Внедрен `golangci-lint` для исключения ошибок и некрасивого кода.
* **CI/CD:** GitHub Actions дял автоматизация тестов, линтинга и сборки образов.
* **Бизнес-логика:** Оптимизированы запросы к БД и переработаны основные сценарии.
* **Тестирование:** Покрытие unit-тестами с моками бизнес-сценариев >70%.

## 🚀 Быстрый старт

### Требования
* Установленный **Go**
* **Docker** и **Docker Compose**

### Установка и запуск
1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/N1ktarchik/BOARD_OF_ISSUES
    cd BOARD_OF_ISSUES
    ```
2.  **Запустите проект:**
    ```bash
    make up
    ```
    *Эта команда поднимет контейнеры с Go-сервером и базой данных PostgreSQL.*

3.  **Генерация Swagger (при изменении аннотаций):**
    ```bash
    make swagger-gen
    ```

## 🛡 Качество кода и CI/CD

В проекте настроен автоматический пайплайн проверки (**GitHub Actions**):
* **Linter:** При каждом пуше запускается `golangci-lint` (v1.64.8) с настроенным конфигом. Проверяется корректность работы с `context`, обработка ошибок в `defer` и отсутствие неиспользуемого кода.
* **Tests:** Автоматический прогон Unit-тестов.
* **CD:** При создании тега `v*.*.*` происходит автоматическая сборка и пуш образа в Docker Hub.

## 📖 Документация API

После запуска сервера интерактивная документация доступна по адресу:
📍 **URL:** `http://localhost:8080/swagger/`

**Основные возможности API:**
* **Пользователи:** Регистрация, логин и редактирование профиля.
* **Доски (Desks):** Создание, подключение по паролю и управление общими досками.
* **Задачи (Tasks):** Полный CRUD, пагинация (page/limit) и фильтрация по статусу.
