# 📋 Board of Issues 2.0

Добро пожаловать во вторую версию **Board of Issues**. Эта версия представляет собой полный архитектурный рефакторинг: переход от монолитной структуры к модульной «чистой архитектуре» (Clean Architecture) удовлетворяющей принципам SOLID.

**Цель рефакторинга:** Масштабируемость, типизация и документация API уровня production.

## 🛠 Технологический стек

* **Язык:** Go (Golang) 1.22+
* **База данных:** PostgreSQL + `pgx` (высокая производительность и контроль над запросами)
* **Роутер:** `gorilla/mux`
* **Логирование:** `slog` (структурированные логи для удобного мониторинга)
* **Документация:** Swagger (OpenAPI 2.0) через `swag`
* **Миграции:**	golang-migrate/migrate	(управление схемой БД)
* **Аутентификация** JWT (JSON Web Tokens)
* **Тестирование** testing + mockgen (Unit-тесты с автоматической генерацией моков)
* **Контейнеризация:** Docker & Docker Compose

## 🏗 Что изменилось в архитектуре (v2 vs v1)

В отличие от первой версии, в **v2** внедрено строгое разделение ответственности:

* **Domain Layer:** Чистая бизнес-логика и сущности. Никакой зависимости от внешних библиотек.
* **Transport Layer (HTTP):** Изолированная обработка HTTP-запросов и ответов.
* **DTO (Data Transfer Objects):** Входящие данные теперь проходят строгую валидацию и типизацию перед тем, как попасть в сервис.
* **Repository Pattern:** Работа с БД скрыта за интерфейсами. Используется пул соединений `pgx` для эффективности.
* **Централизованная обработка ошибок:** Все ответы API имеют единый формат, что упрощает жизнь фронтенд-разработчикам.

* Так же добавлены  **Миграции**, **Документация** (`swagger`), **Логирование**, **Makefile**.

* Улучшена **бизнес логика** и **оптимизированны запросы** к БД

* Покрытие **unit-тестами** с моками бизнес сценариев >70%

## 🚀 Быстрый старт

### Требования
* Установленный Go
* Docker и Docker Compose

### Установка и запуск
1.  **Клонируйте ветку:**
    ```bash
    git clone https://github.com/N1ktarchik/BOARD_OF_ISSUES
    cd BOARD_OF_ISSUES
    ```

2.  **Запустите проект:**
    ```bash
    make run
    ```
    *Эта команда поднимет контейнеры с Go-сервером и базой данных PostgreSQL.*

3.  **Генерация Swagger (при изменении аннотаций):**
    ```bash
    make swagger-gen
    ```

## 📖 Документация API

После запуска сервера интерактивная документация доступна по адресу:
📍 **URL:** `http://localhost:8080/swagger/`

**Основные возможности API:**
* **Пользователи:** Регистрация, логин и редактирование профиля.
* **Доски (Desks):** Создание, подключение по паролю и управление общими досками.
* **Задачи (Tasks):** Полный CRUD, пагинация (page/limit) и фильтрация по статусу выполнения.
