# 📋 Board of Issues — API для управления задачами

**Board of Issues** — это учебный pet-проект, который я писал, чтобы прокачаться в Go: слоистая архитектура, JWT, тесты, работа с PostgreSQL. Но в процессе я понял, что штука получается полезной не только для практики, но и в реальной жизни — как для семьи, так и для небольших команд разработки.

## 🛠 Стек технологий (Tech Stack)

* **Язык:** Go (Golang)
* **Маршрутизация:** Чистая стандартная библиотека (`net/http`) с использованием паттерна **ServerMux** — минимум зависимостей, максимум контроля.
* **Архитектура:** Четкое разделение на слои (Layered Architecture):
  * **Транспорт (`internal/transport`):** Обработка HTTP-запросов, валидация, отправка ответов.
  * **Бизнес-логика (`internal/service`):** Основные правила и сценарии использования приложения.
  * **Репозиторий (`internal/repo`):** Абстракция для работы с базой данных (PostgreSQL).
* **База данных:**
  * **PostgreSQL** — основная БД.
  * **Драйвер `pgx`** — работа с БД.
* **Аутентификация и безопасность:**
  * **JWT (JSON Web Tokens)** — для защиты эндпоинтов и идентификации пользователей.
  * **Bcrypt** — хеширование паролей пользователей и паролей досок.
* **Тестирование:**
  * Реализованы тесты для **хендлеров (handlers)**, **транспортного слоя** и **слоя репозитория** для гарантии целостности данных.
* **Конфигурация:** Переменные окружения (файл `.env`).

## 🤔 А зачем вообще такое нужно?

Идея простая: сделать инструмент, куда можно пригласить **своих людей** и вместе вести какие-то дела. Чтобы было понятно, кто что должен сделать, кто за что отвечает, и чтобы ничего не терялось.

**Для семьи или компании друзей:**
- Семья планирует ремонт — завёл доску, раскидал задачи, все видят, что уже сделано, а что ещё нет.
- Соседи по квартире — кто сегодня моет посуду, кто выносит мусор и покупает продукты.
- Друзья собираются в поход — распределили снаряжение, еду, транспорт.

**Для командной разработки:**
- Небольшая команда разрабатывает проект — создали доску, поставили задачи, назначили ответственных, следим за прогрессом.
- Open source — контрибьюторы видят список issues, берут в работу, отмечают выполненное.
- Pet-проекты вдвоём — удобно синхронизироваться, кто что делает.

То есть это такая **универсальная "доска дел"**, где у всех есть доступ, всё прозрачно, и никто ничего не забывает.

## Что внутри?

- **Доски** — заводи под разные сферы: "Ремонт", "Бэкенд", "Фронтенд", "Путешествие".
- **Участники** — добавляй своих людей, и они сразу видят все задачи.
- **Права доступа** — создатель доски может передать управление другому.
- **Задачи со статусами** — сделал — отметил, все видят прогресс.
- **Фильтры** — покажи только мои задачи или только выполненные.

## 🚀 Функциональность и команды

API построен на принципах REST. Подробная спецификация с примерами доступна по корневому маршруту `/`.

### 👤 Пользователи (Authentication & Profile)
* **Регистрация:** `POST /register` — создание нового аккаунта.
* **Вход:** `POST /login` — получение JWT-токена для дальнейшей работы.
* **Управление профилем (требуют `Bearer Token`):**
  * `PATCH /api/users/name` — смена имени.
  * `PATCH /api/users/email` — смена email.
  * `PATCH /api/users/password` — смена пароля (с подтверждением старого).
  * `POST /api/users` — добавление пользователя на доску (по `desk_id`).

### 📊 Доски (Desks)
* **Создание:** `POST /api/desks` — новая доска с паролем.
* **Получение:** `GET /api/desks` — список всех досок текущего пользователя.
* **Управление (только владелец):**
  * `PATCH /api/desks/{id}/name` — переименование.
  * `PATCH /api/desks/{id}/password` — смена пароля доски.
  * `PATCH /api/desks/{id}/owner` — передача прав владельца.
  * `DELETE /api/desks/{id}` — удаление доски.

### ✅ Задачи (Tasks)
* **Базовые операции:**
  * `POST /api/tasks` — создание задачи на конкретной доске.
  * `GET /api/tasks/{deskId}` — получение всех задач доски.
  * `DELETE /api/tasks/{id}` — удаление задачи.
* **Фильтрация:** `GET /api/tasks?done=true&desk_id=123` — выборка задач по статусу выполнения.
* **Редактирование:**
  * `PATCH /api/tasks/{id}/complyte` — отметить как выполненную.
  * `PATCH /api/tasks/{id}/time` — добавить часы к задаче (`{"hours": 2}`).
  * `PATCH /api/tasks/{id}/description` — изменить описание.

# ⚙️ Установка и запуск (локально)

1. **Клонируй репозиторий:**
   ```bash
   git clone https://github.com/N1ktarchik/BOARD_OF_ISSUSES.git
   cd BOARD_OF_ISSUSES
   ```

2. **Настрой базу данных PostgreSQL** и создай файл `.env` в корне проекта (смотри структуру переменных в коде).

3. **Создай таблицы в БД.** Скрипт инициализации находится в `sql/queries.sql` (или выполни вручную).

4. **Собери и запусти приложение:**
   ```bash
   go run cmd/app/main.go
   ```
   Сервер будет доступен по адресу http://localhost:8080.

# 📚 Примеры использования (cURL)

### 1. Регистрация нового пользователя
```bash
curl -X POST http://localhost:8080/register \
-H "Content-Type: application/json" \
-d '{"name":"Иван","email":"ivan@mail.com","password":"123456"}'
```

### 2. Вход в систему (получаем токен)
```bash
curl -X POST http://localhost:8080/login \
-H "Content-Type: application/json" \
-d '{"email":"ivan@mail.com","password":"123456"}'
```
*В ответе придет JWT-токен. Сохрани его — он понадобится для всех следующих запросов.*

### 3. Создание новой доски (с токеном)
```bash
curl -X POST http://localhost:8080/api/desks \
-H "Authorization: Bearer <вставь_свой_токен>" \
-H "Content-Type: application/json" \
-d '{"name":"Мой проект","password":"board_pass"}'
```

### 4. Получение списка всех досок пользователя
```bash
curl -X GET http://localhost:8080/api/desks \
-H "Authorization: Bearer <вставь_свой_токен>"
```

### 5. Создание задачи на доске
```bash
curl -X POST http://localhost:8080/api/tasks \
-H "Authorization: Bearer <вставь_свой_токен>" \
-H "Content-Type: application/json" \
-d '{"title":"Написать README","description":"Оформить красиво","desk_id":1}'
```

### 6. Отметить задачу как выполненную
```bash
curl -X PATCH http://localhost:8080/api/tasks/1/complyte \
-H "Authorization: Bearer <вставь_свой_токен>"
```

### 7. Удалить задачу
```bash
curl -X DELETE http://localhost:8080/api/tasks/1 \
-H "Authorization: Bearer <вставь_свой_токен>"
```
